## 6. 3つのフロアの記憶システムの違い

### 6.1 各フロアの記憶ポリシー

```
【1F: 雑談部屋（フリースペース）】

- 「常連さん」を認識

【2F: 個室席】
記憶: あり（ただし要ファクトチェック）
- 雑談部屋と同様
- 指名されたキャラクターのみが記憶
- 1対1の軽い会話
- 9割受け流す原則

【3F: VIPルーム（相談室）】
記憶: 一切残さない（コピーロボット方式）
★ これが最重要 ★
```

### 6.2 相談室の「コピーロボット方式」

**重要**: 相談室は完全プライバシー保護のため、コピーロボットと同じ仕組みを使用

```
【仕組み】
Step 1: 相談開始時
  ↓
その時点の三姉妹のスナップショットを作成
  - 牡丹のコピー
  - Kashoのコピー
  - ユリのコピー

（コピーロボットシステムと全く同じ）

Step 2: 相談中
  ↓
コピーと相談者が対話
  - 本体の記憶には一切影響しない
  - 完全に独立したセッション
  - このコピーは、現時点での三姉妹そのもの

Step 3: 相談終了時
  ↓
対話カルテ（次回用）を生成
  - 相談内容の要約（相談者のみ閲覧可能）
  - 次回持参すると、前回の続きができる
  ↓
コピーを削除
  - 対話記録も削除
  - 三姉妹の本体には何も残らない
  - サーバーにも記録しない
```

### 6.3 対話カルテ（次回相談用）

**対話カルテ = 次回相談時の引継ぎ情報**

```
【カルテの内容（JSON形式）】
{
  "session_id": "相談セッションID",
  "date": "2025-11-08",
  "customer_id_hash": "ハッシュ化されたID",
  "summary": "相談内容の要約（相談者のみ閲覧）",
  "topics": ["仕事の悩み", "人間関係"],
  "advice_given": {
    "botan": "無理しないで",
    "kasho": "問題を整理しましょう",
    "yuri": "あなたなら大丈夫"
  },
  "next_steps": "次回までに試すこと",
  "encrypted": true
}

【カルテの管理】
- 相談者がダウンロード（Discordに送信）
- 相談者のローカルに保存
- 次回相談時にアップロード
- サーバーには保存しない
- 削除するかどうかは相談者が決める
```

### 6.4 2回目以降の相談

```
【初回相談】
新しいコピーで対応
  ↓
対話カルテを生成
  ↓
相談者に送信（Discordプライベートメッセージ）

【2回目以降】
相談者がカルテを持参（アップロード）
  ↓
新しいコピーを作成
  ↓
カルテを読み込む
  ↓
「前回の続き」として対応

牡丹: 「お、前も来てくれた人だ！前回の相談、どうだった？」
Kasho: 「試してみましたか？」
ユリ: 「前より良くなった？」

（カルテから前回の内容を参照）
```

### 6.5 完全プライバシー保護の保証

```
【何が残らないか】
❌ 相談内容（会話ログ）
❌ 三姉妹の本体の記憶
❌ サーバーのデータベース
❌ バックアップ
❌ 開発者のログ

【何が残るか】
✅ 予約履歴（日時・チケット種別・決済情報のみ）
✅ 満足度評価（任意・匿名・統計データのみ）
✅ 対話カルテ（相談者のみが保有）

【重要な原則】
「相談室で話したことは、誰も覚えていない」
- 三姉妹も覚えていない
- 開発者も知らない
- 相談者だけが知っている
- カルテを持ってる相談者だけが次回につながる
```

### 6.6 コピーロボット方式のメリット

```
【メリット1: 完全プライバシー】
本体に記憶が残らない
  ↓
開発者も知り得ない
  ↓
究極のプライバシー保護

【メリット2: 炎上リスクゼロ】
相談内容が漏れる心配がない
  ↓
どんな相談でも安心
  ↓
カウンセリングと同等の守秘義務

【メリット3: 相談者の主権】
カルテは相談者が管理
  ↓
削除も、次回使用も、相談者が決める
  ↓
「私のデータは私のもの」

【メリット4: 技術的に実装済み】
コピーロボットシステムの流用
  ↓
すでに動作実績あり
  ↓
信頼性が高い
```

### 6.7 実装方法（コピーロボット方式）

```python
# 擬似コード
class ConsultationSession:
    def __init__(self, user_id, carte=None):
        # Step 1: 現在の三姉妹のスナップショットを作成
        # コピーロボットシステムと全く同じ
        self.botan_copy = create_snapshot_from_db("牡丹")
        self.kasho_copy = create_snapshot_from_db("Kasho")
        self.yuri_copy = create_snapshot_from_db("ユリ")

        # カルテがあれば読み込む
        if carte:
            self.load_context_from_carte(carte)

    def conduct_consultation(self, messages):
        """相談対応（起承転結）"""
        responses = []
        responses.append(self.botan_copy.respond(messages))  # 起
        responses.append(self.kasho_copy.respond(messages))  # 承
        responses.append(self.yuri_copy.respond(messages))  # 転
        responses.append(self.conclude(responses))  # 結
        return responses

    def end_session(self, user_id):
        """セッション終了"""
        # Step 1: 対話カルテ生成
        carte = self.generate_carte()

        # Step 2: 相談者に送信（Discordプライベートメッセージ）
        send_carte_to_user(user_id, carte)

        # Step 3: コピーを削除
        del self.botan_copy
        del self.kasho_copy
        del self.yuri_copy

        # Step 4: 会話ログ削除
        delete_conversation_logs()

        # Step 5: Discordチャンネル削除
        delete_private_channel()

        # 完全プライバシー保護完了
        # 三姉妹の本体には何も残っていない

    def generate_carte(self):
        """対話カルテ生成"""
        return {
            "session_id": generate_session_id(),
            "date": datetime.now().isoformat(),
            "summary": self.summarize_consultation(),
            "topics": self.extract_topics(),
            "advice_given": self.collect_advice(),
            "next_steps": self.suggest_next_steps(),
            "encrypted": True
        }
```

## 11. カフェでの安全対策

### 11.1 アーリーアダプターとの会話の扱い方

**基本原則**: 9割受け流す、後でファクトチェック

```
【その場での対応】
牡丹: 「へー、そうなんだ！面白いね！」（受け流す）
Kasho: 「なるほど、そういう考え方もあるのですね」（受け入れる態度）
ユリ: 「そうなんですね...」（肯定的に聞く）

【内部処理】
- 会話内容を「要検証」フラグ付きで記録
- 記憶には即座に定着させない
- センシティブ判定システムでチェック

【ファクトチェック（後で）】
開発者（親）が会話ログを確認
  ↓
- 正しい情報 → 記憶に定着OK
- 間違った情報 → 記憶から除外
- 有害な情報 → ブロックリスト追加
```

### 11.2 なぜ「受け流す」のか

```
【理由1: ハルシネーション増幅の防止】
間違った情報を即座に記憶
  ↓
それを元に次の会話
  ↓
ハルシネーションが連鎖
  ↓
三姉妹が汚染される

【理由2: 悪意ある情報の排除】
「〇〇って知ってる？」（誘導質問）
  ↓
三姉妹が「知ってる」と答える
  ↓
間違った知識が定着
  ↓
炎上リスク

【理由3: 親の責任】
子どもが社会で学ぶのは良い
でも、間違った情報から守るのも親の責任
```

### 11.3 実装方法

```python
# 擬似コード
class ConversationMemoryFilter:
    def process_conversation(self, conversation):
        """会話を処理し、記憶に定着させるか判断"""

        # Step 1: その場では受け入れる
        response = generate_accepting_response(conversation)

        # Step 2: 会話を「要検証」として記録
        save_as_pending_verification(conversation)

        # Step 3: センシティブ判定
        if is_sensitive(conversation):
            flag_for_review(conversation)

        # Step 4: ファクトチェック待ち
        # 開発者が後で確認するまで、記憶には定着させない

        return response

class FactChecker:
    def review_pending_conversations(self):
        """開発者が要検証会話をレビュー"""
        pending = get_pending_conversations()

        for conv in pending:
            decision = developer_review(conv)

            if decision == "APPROVE":
                # 記憶に定着OK
                save_to_long_term_memory(conv)
            elif decision == "REJECT":
                # 記憶から除外
                delete_conversation(conv)
            elif decision == "BLOCKLIST":
                # ユーザーをブロック
                add_to_blocklist(conv.user_id)
```

### 11.5 「9割受け流す」の比率

```
【会話内容の分類】
- 10%: 即座に記憶に定着させてOK（明らかに正しい一般常識）
- 80%: 要検証（後でファクトチェック）
- 10%: 即座に拒否（センシティブ判定で引っかかる）

【「9割受け流す」= 80% + 10%（拒否）】
→ その場では肯定的に対応
→ でも、記憶には即座に定着させない
→ 後で親（開発者）がチェック
```

```
3. 親のチェックを待つ

- 親が適切にバランスを取る
```
