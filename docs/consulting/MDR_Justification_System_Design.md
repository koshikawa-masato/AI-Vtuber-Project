## 1. 概要

### 1.1 システムの目的

米国FDA MDR（Medical Device Reporting）報告における正当性文章（Justification Statement）の作成を支援するシステム。過去事例をベースとした「検索・置換」アプローチにより、ハルシネーションを排除し、安定した品質の文書を効率的に作成する。

### 1.2 対象業務

| 項目 | 内容 |
|------|------|
| 対象部門 | 内視鏡事業部 RA（Regulatory Affairs）部門 |
| 対象業務 | MDR報告要否判断の正当性文章作成 |
| 現状工数 | 1件あたり2〜10分（平均6分） |
| 処理件数 | 1日5〜10件 |
| 目標 | 1件あたり1〜2分（70-80%削減） |

### 1.3 設計思想

```
【重要原則】
AIは「生成」しない。「検索」と「置換」のみ行う。
判断と承認は必ず人間が行う。
すべての出力に引用元を明示する。
```

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|

## 3. データベース設計

### 3.1 テーブル構成

```sql
-- 過去事例テーブル
CREATE TABLE past_cases (
    id SERIAL PRIMARY KEY,
    case_id VARCHAR(50) UNIQUE NOT NULL,      -- 事例ID（例: MDR-2024-001）
    product_name VARCHAR(200) NOT NULL,        -- 製品名
    product_code VARCHAR(50),                  -- 製品コード
    complaint_category VARCHAR(100),           -- 苦情カテゴリ
    complaint_summary TEXT NOT NULL,           -- 苦情要約
    mdr_required BOOLEAN NOT NULL,             -- MDR要否（TRUE/FALSE）
    justification_ja TEXT NOT NULL,            -- 正当性文章（日本語）
    justification_en TEXT NOT NULL,            -- 正当性文章（英語）
    decision_date DATE NOT NULL,               -- 判断日
    reviewer VARCHAR(100),                     -- 判断者
    embedding vector(1536),                    -- ベクトル埋め込み
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- インデックス
CREATE INDEX idx_past_cases_embedding
    ON past_cases USING ivfflat (embedding vector_cosine_ops);
CREATE INDEX idx_past_cases_product ON past_cases (product_name);
CREATE INDEX idx_past_cases_category ON past_cases (complaint_category);

-- 用語集テーブル
CREATE TABLE glossary (
    id SERIAL PRIMARY KEY,
    term_ja VARCHAR(200) NOT NULL,             -- 日本語用語
    term_en VARCHAR(200) NOT NULL,             -- 英語用語
    category VARCHAR(100),                     -- カテゴリ（製品名/技術用語/規制用語）
    notes TEXT,                                -- 備考
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 作業履歴テーブル
CREATE TABLE work_history (
    id SERIAL PRIMARY KEY,
    complaint_id VARCHAR(50) NOT NULL,         -- 苦情ID
    product_name VARCHAR(200) NOT NULL,        -- 製品名
    complaint_summary TEXT NOT NULL,           -- 苦情要約
    selected_case_id VARCHAR(50),              -- 選択した過去事例ID
    similarity_score FLOAT,                    -- 類似度スコア
    justification_ja TEXT NOT NULL,            -- 生成した正当性文章（日本語）
    justification_en TEXT NOT NULL,            -- 生成した正当性文章（英語）
    mdr_required BOOLEAN NOT NULL,             -- MDR要否判断
    operator VARCHAR(100) NOT NULL,            -- 作業者
    approved_by VARCHAR(100),                  -- 承認者
    status VARCHAR(20) DEFAULT 'draft',        -- ステータス（draft/approved）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    approved_at TIMESTAMP
);

-- 監査ログテーブル
CREATE TABLE audit_log (
    id SERIAL PRIMARY KEY,
    user_id VARCHAR(100) NOT NULL,
    action VARCHAR(50) NOT NULL,               -- 操作（search/generate/approve等）
    target_table VARCHAR(50),
    target_id INTEGER,
    details JSONB,                             -- 詳細情報
    ip_address VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 初期データ投入

既存Excel 3年分のデータを投入：

```python
# データ投入スクリプト（概要）
import pandas as pd
from openai import OpenAI

def import_excel_to_db(excel_path: str):
    """Excelから過去事例をインポート"""
    df = pd.read_excel(excel_path)

    for _, row in df.iterrows():
        # 1. 苦情要約をベクトル化
        embedding = openai_client.embeddings.create(
            model="text-embedding-3-small",
            input=row['complaint_summary']
        ).data[0].embedding

        # 2. DBに挿入
        insert_past_case(
            case_id=row['case_id'],
            product_name=row['product_name'],
            complaint_summary=row['complaint_summary'],
            justification_ja=row['justification_ja'],
            justification_en=row['justification_en'],
            mdr_required=row['mdr_required'],
            embedding=embedding
        )
```

## 6. AI利用ポリシー

### 6.1 AIの役割と制限

| AIが行うこと | AIが行わないこと |
|-------------|-----------------|
| 類似事例の検索・スコアリング | MDR要否の最終判断 |
| テンプレートへの値置換 | 正当性文章の「創作」 |
| 用語集に基づく英訳 | 規制解釈 |
| 形式チェック | 承認 |
