##### 創作哲学の全体像

**開発者の手法は、全ての創作活動で一貫している:**

```
1. 既存のものを深く観察する
2. 本質的な特徴を抽出する（似顔絵のように）
3. 自分なりに再構築する（完全コピーではなく、理解した上での再創造）
```

##### 3つの創作分野での実践

**音楽（FM音源 + エレキベース）:**
- FM音源（1980年代の音色、レトロ資産）の特徴を理解
- ＋ エレキベース実演奏（自分の技術、身体性）
- ＝ **私なりの再構築**

**AiCO（牡丹プロジェクト）:**
- パソコン通信時代の対話（1970-80年代、レトロ資産）の本質を抽出
- ＋ 現代のLLM技術（自分の50年の経験、育成思想）
- ＝ **牡丹という私なりの再構築**

**組み込みプログラミング:**
- 既存アルゴリズム、データシート、実装基板、回路図から特徴を抽出
- ＋ 制約条件の中で最適化
- ＝ **その環境専用の再実装（車輪の再発明）**

##### 組み込みでの「答えの出し方」

**開発者の言葉:**
> 「組み込み系のことは、だいたい理解ができていて、既存のソースか、データシート、そして実装基板、回路図だけで答えが出る」

**4つの情報源から特徴抽出:**
1. **既存ソース** = 実装パターン
2. **データシート** = ICの本質的特徴（タイミング、電気特性）
3. **実装基板** = 物理的制約
4. **回路図** = システムの構造的特徴

**これも「似顔絵」と同じ:**
- 特徴を正確に抽出できれば、答えが出る
- 不安ではなく、確信を持って実装できる

**2025年10月29日以降:**

- **ブレない軸を獲得した**
